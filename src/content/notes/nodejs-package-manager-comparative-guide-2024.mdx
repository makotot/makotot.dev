[2024 Node.js Package Manager Guide: npm, Yarn, pnpm Compared](https://nodesource.com/blog/nodejs-package-manager-comparative-guide-2024/) を読んで、メモ。

- 2024年現在、Node.jsプロジェクトのパッケージマネージャーとしてnpm、yarn、pnpmがフロントランナーとして存在している。
  - それぞれの違い、メリット、デメリットを紹介する記事。
- npm
  - Node.jsにおけるデフォルトのパッケージマネージャー。
  - 良い点
    - 比類のない膨大なパッケージレジストリー。既存の豊富なソリューションを活用できる。
    - デフォルトでNode.jsにバンドルされている。Node.jsのランタイムとの互換性が保証されている。
    - 成熟したエコシステムがある。
    - 包括的なCLIが提供されている。
    - セマンティックバージョニングに基づいている。予期しづらい破壊的な変更を避けやすくなっている。
    - カスタムのスクリプトを実行できる。
    - npmレジストリーとのシームレスな統合。
  - 悪い点
    - パフォーマンス面で問題がある。
    - パッケージのバージョン管理と依存関係の解決が複雑。
    - 依存関係の肥大化。
    - セキュリティーの懸念がある。
    - 集中管理するレジストリーへの依存。単一障害点になりやすい。
    - オフラインサポートが限定的。
- yarn
  - 良い点
    - パフォーマンス面で優れている。
    - 依存関係の衝突を防ぐロックファイル。
    - 安定したオフラインサポート。
    - 直感的なCLI。
    - 改善されているエラーハンドリング。
    - npmとの高い互換性。
  - 悪い点
    - npmとの互換性に問題が生じることがある。
    - 大きな依存関係を持つ場合、システムリソースを大幅に消費する可能性がある。
    - コミュニティーが小さい。
    - 設定のオプションが少ない。
- pnpm
  - 「Performant npm」
  - 良い点
    - 依存関係の共有。シンボリックリンクによりディスクスペースが節約される。
      - [シンボリックリンクを使用した `node_modules` の構造 | pnpm](https://pnpm.io/ja/symlinked-node-modules-structure)
    - インストール時間の短縮。依存関係の共有やキャッシュの活用による。
    - 依存関係の衝突を防ぐロックファイル。
    - ネットワーク帯域幅の削減。プロジェクト間で共有された依存関係は再利用できるので、ダウンロードが不要。
    - キャッシュ効率の向上。
    - 直感的なCLI。
    - npmとの高い互換性。
  - 悪い点
    - 従来のパッケージマネージャーと異なる部分への学習コスト。
    - npmとの互換性に問題が生じることがある。
    - コミュニティーが小さい。
